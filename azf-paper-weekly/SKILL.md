# AZF 论文周报

## 定位

把本 Skill 当作已安装 `paper-weekly` CLI 的薄编排层。Python 程序负责采集、SQLite、去重、筛选、模型评估、模板渲染和原子写入；本 Skill 只负责识别意图、检查权限、映射命令、做质量门禁和交接。

开始前读取：

- 需要路径、配置字段、命令映射或模型切换时，读取 [references/config-and-paths.md](references/config-and-paths.md)。
- 涉及生成、补跑、计划任务、云端/中转站或精读交接时，读取 [references/safety-and-handoff.md](references/safety-and-handoff.md)。

## 不可突破的边界

- 不下载论文 PDF。
- 不自动写入 Zotero。
- 不运行 MinerU，不自行创建正式精读工作区。
- CLI 永不覆盖已存在的周报；`文件冲突策略` 必须保持 `skip`。只有用户在当前请求中明确要求替换某一周次时，才允许先做项目外回滚备份并核对哈希，再删除唯一目标文件后重新生成；不得把这一例外实现成通用 `overwrite` 策略。
- 真实模型 API Key 只允许明文保存在被 Git 忽略的唯一生产 YAML或 `config/profiles/` 档案；不保存到 Skill、可提交示例、脚本参数、数据库或日志，不读取后回显。
- 配置管理器可在用户明确点击保存或复制时，把含 Key 的配置复制到同样被 Git 忽略的 `config/profiles/`；Skill 自身仍不得读取后显示或把配置复制到其他位置。
- 不复制正式周报模板。模板的唯一事实来源始终是 Obsidian 中的正式模板文件。
- 不把数据库、日志或生产配置放进 Skill 目录。
- 不把本地期刊分区 XLSX 放进 Skill 或 Git；它只保存在项目内被忽略的 `data/local/journal-rankings/`。
- 不直接加载 GGUF、safetensors 或 Hugging Face 模型目录；由本地运行时提供 HTTP 接口。

## 工作流

### 1. 定位并验证

1. 优先使用生产配置默认路径；若路径不存在或用户给出其他配置，先定位并向用户明确当前使用的绝对路径。
2. 运行 `config validate`。正式生成前还要确认模板问题为空、`允许生产写入: true`、至少一个来源启用。
3. 启用 `期刊评价` 时，确认校验结果含本地数据年份、无歧义记录数、名称歧义数且问题列表为空；只接受规范化精确匹配，未收录、预印本和名称歧义使用配置的中性分。
4. 本地 Ollama 模型需确认服务健康且目标模型存在；其他本地运行时需确认接口地址和模型 ID。云端或中转站只确认生产 YAML 的 `API密钥` 已填写或兼容环境变量存在，不得显示密钥值。
5. 只读预演使用 `production-preview` 或 `--dry-run`，不得把预演描述成已生成。

### 2. 根据用户意图执行

- “检查配置/路径/生产准备”：只运行配置验证和生产预演。
- “查看缺失周次”：运行 `gaps`；注意首次运行可能初始化本地运行数据库。
- “生成本周周报/运行计划任务”：处理上一完整 ISO 周。生成属于生产写入，必须有当前请求中的明确授权。
- “生成指定周”：用 `generate --week`。若目标已存在，通常接受 `skipped_existing`；只有用户明确要求替换该周时，才能进入受控重建流程。
- “替换/重建已有周报”：先验证目标绝对路径位于配置的生产周报根目录且 ISO 周次完全一致；把旧报告复制到项目外回滚目录并记录 SHA-256；只删除这一份目标 Markdown，不删除数据库、模板、其他周次或无关日志；再调用普通 `generate --week`，因此程序内部仍保持 `skip`。
- “补跑”：先列出候选周次；只补跑用户明确选择的 ISO 周次，允许不连续选择。正式补跑前再次确认写入授权。
- “校验周报”：运行 `validate --week`，并检查概览与正文标题集合一致、没有占位内容或 PDF 直链。
- “切换本地/自定义模型”：OpenAI-compatible 运行时保持 `provider: openai_compatible`；Ollama 原生 `/api/chat` 使用 `provider: ollama_native`。本机 HTTP 可无密钥，私有局域网 HTTP 必须显式开启 `允许不安全HTTP`；公网 HTTP 永远拒绝。修改后先验证和只读预演。
- “切换云端/中转站”：保持 `provider: openai_compatible`，在被 Git 忽略的生产 YAML 填写 `API密钥`、准确模型 ID 和 HTTPS Base URL；先解除费用安全锁并填写真实单价与正数上限，再做单周预览。`api_key_env` 只作旧配置兼容。
- “可视化维护多个配置”：优先双击项目根目录 `启动论文周报配置管理器.cmd`，命令行场景再调用 `scripts/open-config-manager.ps1`。档案位于 `config/profiles/`，验证通过后才原子切换唯一生产 YAML；GUI 不显示 Key。计划任务通过 `config model-info` 自动决定是否检查 Ollama，无需随档案切换修改任务参数。
- “注册或修改计划任务”：这是外部持久化变更，必须有明确授权。默认任务名为 `AZF Paper Weekly`，每周一 09:00，错过后尽快运行，当前交互用户，受限权限。

可直接运行 [scripts/paper-weekly.ps1](scripts/paper-weekly.ps1) 做安全命令映射。写入类操作必须显式传入 `-ConfirmProductionWrite`。

### 3. 质量门禁

生成后必须：

1. CLI 严格校验返回 `valid` 且问题列表为空。
2. 确认文件位于目标年份目录且周次正确。
3. 比对速览表与正文条目数量、标题一致。
4. 确认没有 PDF 直链、模板占位符或半成品临时文件。
5. 再运行同一生成命令，确认返回 `skipped_existing` 且文件哈希不变。
6. 单一来源失败可以生成，但必须在报告中透明披露；全部来源失败不得生成空周报。
7. 启用期刊评价时，检查入选论文展示中科院升级版年份、分区或中性原因，并在来源与质量说明中报告精确匹配数量；期刊分最多只占 5 分，不能替代内容质量。

### 4. 精读交接

当用户从周报中选中论文并要求阅读、翻译或精读时，停止本 Skill 的生产流程，调用现有 `azf-literature-reading-workflow`。由该工作流负责 Zotero、本地 MinerU、版式复核、高清图、逐句翻译和两轮精读；交接前仍须按其规则定位并确认笔记路径。

## 报告方式

向用户报告实际结果，而不是计划：

- ISO 周次、候选数、入选数、成功/失败来源。
- 本地分区数据年份、无歧义记录数、歧义数，以及入选论文精确匹配/中性处理数量。
- 配置、周报、运行目录和计划任务的绝对路径或名称。
- 校验、幂等性、计划任务上次结果及下次运行时间。
- 本地/云端/中转站只说明接口已就绪或还缺哪些服务、模型或密钥配置，不显示密钥。
