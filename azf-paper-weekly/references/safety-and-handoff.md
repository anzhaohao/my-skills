# 安全、质量与精读交接

## 权限矩阵

无需额外生产写入授权：

- 读取配置或报告。
- `config validate`。
- `production-preview`。
- `generate --dry-run`、`run --scheduled --dry-run`、`backfill --dry-run`。
- 校验既有周报。

需要用户明确授权：

- 首次或任何正式周报生成。
- 指定周次正式补跑。
- 创建或修改生产配置。
- 注册、修改或删除 Windows 计划任务。
- 安装或更新 Skill。
- 切换到可能付费的云端或中转站模型。

授权只覆盖用户指明的动作，不自动扩展到 PDF、Zotero、MinerU 或精读。

## 生成质量门禁

- 目标文件已存在时必须返回 `skipped_existing`，不得重命名后覆盖或删掉重建。
- 写入前必须通过正式模板契约校验，写入使用临时文件加原子替换。
- 单一来源失败可以继续，但报告必须显示失败来源和错误摘要。
- 全来源失败必须停止，不能生成零论文报告掩盖故障。
- 合法空周可以生成零论文报告。
- 检查速览表和正文标题逐项一致；报告不得含模板占位符、PDF 直链或临时文件。
- 生成后复跑同一命令，验证 `skipped_existing` 和哈希不变。

## 已有周报受控重建

默认仍不得覆盖或删除。只有用户在当前请求中明确指定替换某一 ISO 周次时：

1. 先完成配置校验、模型信息、生产预演和一次完整真实预览。
2. 用配置计算目标绝对路径，确认它位于生产周报根目录、周次和文件名都正确；不得按模糊文件名批量删除。
3. 把唯一目标 Markdown 复制到 `E:\software\CodexPlusPlus\Codex备份\YYYYMMDD_HHMMSS_任务名_修改前备份` 并记录 SHA-256。
4. 只删除这一份报告；不删除 SQLite、运行历史、模板、其他周次、配置或无关日志。数据库候选和报告行会由新运行安全更新，不需要破坏性清理。
5. 使用普通 `generate --week` 重新生成，运行严格 `validate`，检查分区证据、标题一致性、来源披露、无 PDF 直链和无临时文件。
6. 再次生成必须返回 `skipped_existing` 且新文件 SHA-256 不变。

这个流程不改变 `文件冲突策略: skip`，也不增加 overwrite 命令。

## 模型与密钥

- 本地 Ollama 健康端点：`http://127.0.0.1:11434/api/tags`。
- 回环地址可使用 HTTP；私有局域网 HTTP 必须由用户显式设置 `允许不安全HTTP: true`。该开关不能放行公网 HTTP。
- 公网 OpenAI-compatible 接口必须使用 HTTPS 与 Bearer API Key。
- 真实模型密钥默认直接写入被 Git 忽略的生产 YAML；内部必须使用 `SecretStr` 遮蔽并排除序列化。`api_key_env` 仅保留兼容。
- 只检查直接密钥是否非空或兼容环境变量是否存在，不输出、记录、复制、备份或回显值。
- 含真实密钥的生产 YAML 不得提交 Git、复制进 Skill、普通归档或聊天；修改前只做不会暴露内容的哈希级回滚记录，除非用户明确要求复制密钥文件。
- 受版权约束的本地期刊分区 XLSX 只保存在项目内 Git 忽略目录；Skill、Git、普通发布包和聊天中只记录路径、年份、来源说明、记录数和 SHA-256，不复制表格正文。
- 配置管理器的档案目录是唯一例外：用户明确点击保存/复制后，可在 `config/profiles/` 内维护多个含 Key 的 `.local.yaml`；目录必须保持 Git 忽略，GUI 只能显示密钥是否已配置。
- 档案切换必须先严格验证，再原子替换唯一生产 YAML；配置管理器不得调用真实模型、生成周报或修改模板。
- GGUF、safetensors 等本地模型文件只能由 Ollama、LM Studio、llama.cpp、vLLM、LocalAI、Xinference 等运行时加载；Skill 和 CLI 不直接加载文件。
- 第三方中转站会接收论文题名、摘要和元数据；提醒用户自行确认隐私、留存和计费政策。
- 费用安全锁为 0 时不得为了“试试看”擅自调大；需要用户明确选择模型和预算。

## 精读交接

周报只完成摘要级筛选。用户选择论文并要求继续后：

1. 记录用户选择的论文题名、DOI/arXiv 页面、入选理由和目标阅读深度。
2. 调用 `azf-literature-reading-workflow`。
3. 由该工作流定位 Zotero 条目、PDF、MinerU 解析产物和 Obsidian 路径，并按其规则先确认路径。
4. 由该工作流执行两轮解析、版式复核、高清图、逐句翻译和深读笔记。
5. 本 Skill 不自行下载 PDF、不写 Zotero、不启动 MinerU，也不复制其模板或目录结构。
