# 逐句精读 skill

这个 skill 用来把论文段落，尤其是摘要，整理成 Obsidian 里的逐句精读笔记。

# 什么时候触发

当我说“对这篇文献进行精读”“逐句精读论文”“生成论文精读笔记”“把摘要逐句拆解”等类似需求时，应触发这个 skill。

# 当前保存的规则

- 先确认论文根目录和 `逐句精读` 文件夹位置；没有就新建。
- 摘要也算一个自然段，一个自然段生成一个精读笔记。
- 每个句子卡片前必须有句子级标题，格式为 `## 第 N 句：简明小标题`。
- `第 N 句` 负责定位，小标题负责概括本句作用或核心意思，例如 `提出核心测量对象`、`说明trace可直接判读`。
- 每句使用“中英文对照”非嵌套 callout。
- `【英文原文】` 留空，下方保留 PDF++ 句级 `selection=` 跳转链接，由 PDF 承载原文。
- 专业术语只建理解当前段落必需的核心项，放到 `扫盲班` 文件夹。
- 不在句子块开头放“本句功能”，段落功能放到“本段犀利总结与定位”。
- 保持工程实战、零基础友好、剥洋葱式解释。
